# Code of Conduct

## Our Pledge
We are committed to making collaboration respectful, efficient, and safe for everyone in this organization.

---

## Expected Behavior

All contributors are expected to:

- Communicate respectfully and professionally
- Provide constructive feedback
- Focus on technical issues, not personal opinions
- Respect differing viewpoints and experience levels
- Follow agreed engineering standards and processes

---

## Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment or offensive comments
- Disrespectful or aggressive communication
- Personal attacks or discrimination of any kind
- Deliberate disruption of workflows
- Ignoring security or process rules intentionally

---

## Engineering Standards

All contributions must follow:

- Repository-specific rules
- Organization-wide `DEVELOPMENT_STANDARDS.md` (if present)
- Security policies defined in `SECURITY.md`

---

## Decision Making

- Technical decisions should be based on:
  - Maintainability
  - Security
  - Performance
  - Clarity

- Blocking disagreements should be escalated to maintainers.

---

## Enforcement

Maintainers may:

- Request changes to contributions
- Reject non-compliant pull requests
- Temporarily restrict participation for repeated violations

---

## Scope

This Code of Conduct applies to:

- All repositories in this organization
- Issues, pull requests, discussions
- Internal and external contributors
